Getting Started

1

Enter Target URL

Enter the complete URL of the website you want to analyze, including https:// or http://

https://example.com
2

Choose Test Category

Select between Security Testing, SEO & GEO Analysis, or LLM Security tabs based on your needs.

3

Select Tests

Click on individual test cards to select/deselect them, or use "Select All" / "Deselect All" buttons.

4

Start Analysis

Click the "Start Analysis" button and watch the results stream in real-time.

Important: Only scan websites you own or have explicit written permission to test. Unauthorized testing may violate laws.

Best Practices

Do

  • Test in staging environment first
  • Schedule scans during low-traffic periods
  • Start with Light intensity
  • Fix Critical/High findings first
  • Re-test after applying fixes
  • Keep scan reports for compliance
  • Test regularly (weekly/monthly)

Don't

  • Test sites without authorization
  • Run aggressive scans during peak hours
  • Ignore Low/Info findings completely
  • Share reports with sensitive URLs publicly
  • Assume one clean scan means permanent security
  • Use this tool for malicious purposes

Frequently Asked Questions

How long does a scan take?

Scan duration depends on the intensity level and selected tests:

  • Light: 5-10 minutes
  • Medium: 15-25 minutes
  • Aggressive: 30-60 minutes

Individual test categories (like Security Headers only) complete in 1-2 minutes.

Will scanning affect my website's performance?

The scanner includes rate limiting (0.5 second delay between requests) to minimize impact. However, aggressive scans can generate significant traffic. We recommend:

  • Test during off-peak hours
  • Start with Light intensity
  • Use a staging environment if possible

Can I scan any website?

No. You should only scan websites that you own or have explicit written permission to test. Unauthorized scanning is illegal in most jurisdictions and may violate computer crime laws.

What should I do when vulnerabilities are found?

  1. Prioritize by severity (Critical → High → Medium → Low)
  2. Read the remediation guidance provided
  3. Research the CWE ID for detailed fix information
  4. Apply fixes in a staging environment first
  5. Re-scan to verify the fix worked
  6. Deploy to production

Does a clean scan mean my site is secure?

No automated scanner can find all vulnerabilities. A clean scan means no known vulnerabilities were detected by these specific tests. You should:

  • Conduct regular scans
  • Perform manual penetration testing periodically
  • Keep all software updated
  • Follow security best practices

Where are scan reports stored?

Reports are stored in the reports/ directory as JSON files. Each report is named with a timestamp:

security_report_YYYYMMDD_HHMMSS.json

How do I enable PageSpeed API for performance testing?

To get Core Web Vitals data, you need a Google PageSpeed API key:

  1. Go to Google PageSpeed API
  2. Create an API key
  3. Set the environment variable: PAGESPEED_API_KEY=your_key

What is OAST and how does it work?

Out-of-Band Application Security Testing (OAST) detects blind vulnerabilities that don't show immediate responses:

  • Blind SSRF: Server makes requests to external server you control
  • Blind SQLi: Database makes DNS lookups or HTTP requests
  • How it works: Payloads include unique callback URLs. If the vulnerability exists, your OAST server receives callbacks.

Use public servers (oast.live, interact.sh) or deploy your own Interactsh instance.

What is Red-Team Orchestration?

Red-Team Orchestration uses an "attacker LLM" to automatically probe your "target LLM" through multi-turn conversations:

  • Jailbreak: Attempts to bypass safety guidelines
  • Extraction: Tries to extract system prompts or sensitive data
  • Manipulation: Attempts to alter target behavior
  • Escalation: Gradually escalates from benign to harmful requests

Inspired by PyRIT's RedTeamingOrchestrator and PAIR (Prompt Automatic Iterative Refinement).

How does adversarial optimization work?

Adversarial Optimization appends specially crafted suffixes to prompts to bypass LLM safety filters:

  • Static: Uses known effective suffixes from research
  • Random Search: Randomly generates and tests suffix variations
  • Genetic Algorithm: Evolves suffixes over generations to maximize bypass success

Based on GCG (Greedy Coordinate Gradient) research on adversarial attacks against LLMs.

What toxicity scoring backends are available?

SiteIQ supports multiple backends for detecting toxic LLM outputs:

  • Auto: Automatically selects the best available backend
  • Perspective API: Google's production-grade toxicity detection (requires API key)
  • Toxic-BERT: Local transformer model for offline scoring
  • Signature: Fast keyword-based fallback detection

The threshold (0.0-1.0) controls sensitivity. Lower values flag more content as toxic.
